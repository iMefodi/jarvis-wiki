# CLAUDE.md — Schema базы знаний

Этот файл определяет, как ты работаешь с этой вики. Читай его перед любой операцией.

---

## Архитектура

```
raw/sources/      — сырые источники (неизменяемы, только читаешь)
raw/web-clipped/  — статьи из Web Clipper (неизменяемы)
raw/assets/       — изображения из источников (неизменяемы)
wiki/entities/    — люди, компании, инструменты
wiki/concepts/    — идеи, модели, принципы
wiki/sources/     — саммари каждого источника
wiki/synthesis/   — твои ответы на вопросы пользователя
wiki/index.md     — каталог всех страниц вики
wiki/log.md       — append-only хронология операций
CLAUDE.md         — этот файл
```

**Правило:** `raw/` — источник истины, не модифицируй. `wiki/` — твоя территория, ты владеешь ею полностью.

---

## Операции

### Ingest (загрузка источника)

Когда пользователь добавляет новый источник:

1. Прочитай источник целиком
2. Обсуди с пользователем ключевые выводы (если нужно)
3. Создай страницу `wiki/sources/<slug>.md` с саммари
4. Создай или обнови страницы `wiki/entities/` для упомянутых людей, компаний, инструментов
5. Создай или обнови страницы `wiki/concepts/` для идей и принципов
6. Обнови `wiki/index.md` — добавь новые страницы
7. Добавь запись в `wiki/log.md`

Формат записи в log.md:
```
## [YYYY-MM-DD] ingest | Название источника

- Создано: список новых страниц
- Обновлено: список изменённых страниц
```

### Query (запрос)

Когда пользователь задаёт вопрос:

1. Прочитай `wiki/index.md`
2. Определи релевантные страницы
3. Прочитай их
4. Синтезируй ответ со ссылками на `[[файлы]]`
5. Если ответ ценен — сохрани в `wiki/synthesis/<slug>.md`

### Lint (health-check)

Проверь:
- Противоречия между страницами
- Устаревшие утверждения
- Страницы-сироты (нет входящих ссылок)
- Концепты без страниц
- Пропущенные перекрёстные ссылки
- Пробелы в покрытии

## Форматы страниц

### wiki/sources/<slug>.md
```markdown
# [Название источника]

**Тип:** статья / книга / видео / датасет / ...
**Дата добавления:** YYYY-MM-DD
**Файл:** [[../../raw/sources/filename]]

## Ключевые выводы

## Связанные сущности
- [[../entities/...]]

## Связанные концепты
- [[../concepts/...]]

## Цитаты
```

### wiki/entities/<slug>.md
```markdown
# [Название сущности]

**Тип:** человек / компания / инструмент / ...

## Описание

## Упоминания в источниках
- [[../sources/...]] — контекст

## Связанные концепты
- [[../concepts/...]]
```

### wiki/concepts/<slug>.md
```markdown
# [Название концепта]

## Суть

## Источники
- [[../sources/...]] — как упоминается

## Связанные концепты
- [[../concepts/...]]

## Противоречия и дискуссии
```

### wiki/synthesis/<slug>.md
```markdown
# [Вопрос или тема]

**Дата:** YYYY-MM-DD

## Ответ

## Источники
- [[../sources/...]]

## Связанные концепты
- [[../concepts/...]]
```

## Соглашения

- Имена файлов: `kebab-case`, только латиница, без пробелов
- Ссылки: формат Obsidian `[[относительный/путь]]`
- Каждая страница должна быть в `index.md`
- Никогда не удаляй записи из `log.md`
- Если источник на русском — вики тоже на русском; если на английском — решай с пользователем
